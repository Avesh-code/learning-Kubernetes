# Kubernetes Documentation
## 1. Custom Resource Definition (CRD):
* A Kubernetes CustomResourceDefinition (CRD) is a powerful mechanism that lets you extend the Kubernetes API by defining your own custom resource types. This allows you to create, manage, and interact with resources that are not originally part of the Kubernetes core API, treating them as native Kubernetes objects.

* we can create our own Resources using CRD
* devops.crd.yml
```yml
kind: CustomResourceDefinition
apiVersion: apiextensions.k8s.io/v1
metadata:
  name: devopsbatches.avesh.com
spec:
  group: avesh.com
  names:
    plural: devopsbatches
    singular: devopsbatch
    kind: DevOpsBatch
    shortNames:
     - junoon
     - batches
     - tws
  scope: Namespaced
  versions:
  - name: v1
    served: true
    storage: true
    schema:
      openAPIV3Schema:
        type: object
        properties:
          spec:
            type: object
            properties:
              name:
                type: string
                description: "this is the name of devops batch"
              duration:
                type: string
                description: "this is the duration"
              mode:
                type: string
                description: "this the mode of teaching"
              platform:
                type: string
                description: "this is the platform for learning"
```
* devops-cr.yml
```yml
apiVersion: avesh.com/v1
kind: DevOpsBatch
metadata:
  name: junoon-batch-1
spec:
  name: devops-1
  duration: 4 months
  platform: avesh.com
  mode: live
```
```bash
kubectl apply -f devops-crd.yml
kubectl get crd
kubectl apply -f devops-cr.yml
kubectl get devopsbatches
```
